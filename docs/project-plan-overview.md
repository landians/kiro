# Kiro 项目任务拆解总览

## 1. 文档说明

本文档用于把 Kiro 项目规划整理成“任务拆解 + 完成状态”的执行清单。

- 状态判定以当前仓库中已经存在的代码、路由、migration 和文档为准。
- `[x]` 表示当前仓库已经有明确实现或已经落地。
- `[ ]` 表示当前尚未完成，或只完成了前置部分但还没有形成完整闭环。
- 为避免高估完成度，只有真正已经落地的子任务才会勾选。

## 2. 项目总体定位

Kiro 是一个面向订阅制数字商品或服务的后端基础设施框架，目标是在有新产品想法时，可以基于统一底座快速搭建出一个具备账号、认证、JWT 鉴权、管理后台、商品、支付、订阅、通知和本地化能力的后端服务。

参考文档：

- [Kiro 技术栈选型](project-technology-stack.md)
- [Kiro 项目开发规范](project-development-stanards.md)

## 3. 当前完成情况概览

按当前代码状态看，项目整体处于“基础底座 + 用户登录闭环 + 后台基础能力 + 商品建模与展示”阶段。

- M0 基础底座：已完成较多，健康检查细化、统一成功响应和系统级集成测试仍未补齐。
- M1 用户与登录：Google 登录、JWT、刷新、登出、用户查询与更新已具备，安全治理和专门的当前用户接口仍未完成。
- M2 管理后台：独立后台服务、管理员密码登录、用户分页与状态管理、商品后台管理已落地，管理员授予和审计能力尚未补齐。
- M3 商品模块：商品与计划的表结构、后台管理接口、前台展示接口已落地，权益模型和购买前校验尚未开始。
- M4 及以后：支付、订阅、邮件、通知、多语言、邀请等模块还未真正开工。
